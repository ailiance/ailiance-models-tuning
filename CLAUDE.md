# CLAUDE.md — ailiance-models-tuning

ML fine-tuning pipeline for the Ailiance ecosystem: CPT / SFT / RLVR on domain-expert LLMs (STM32, KiCad, SPICE, FreeCAD, PlatformIO, EMC, power, DSP, IoT, embedded).

> **Parent context** : `../CLAUDE.md` (F4L monorepo) — workspace rules, machines, deploy, infra. Do not duplicate here.
> **Note** : ce dossier est dans `.gitignore` parent, **pas** un submodule. Clone standalone via `git clone git@github.com:ailiance/ailiance-models-tuning.git`.

## What makes this project different

- **Only GPU-bound project** in F4L — heavy training runs on **KXKM-AI** (RTX 4090 24GB), not GrosMac/Tower
- **Not packaged** — `src/ailiance_tuning/` is a thin lib; scripts/ are the real entry points, invoked directly
- **Artifacts are heavy** — adapters in `outputs/*/`, registry in `artifacts/model_registry.json`, all gitignored
- **10 domains, 1 pipeline** — same trainer, same YAML shape, different datasets + `hub_model_id`

## Pipeline at a glance

```text
datasets/builders/build_<domain>_dataset.py   (seed + HF merge, chat-format JSONL)
  → datasets/processed/<domain>_train.jsonl
  → scripts/validate_dataset.py                (role/content schema check)
  → scripts/train_sft.py --config configs/sft_<domain>.yaml    (QLoRA NF4 4-bit on 4090)
  → outputs/sft-<domain>/adapter_model.safetensors
  → scripts/eval_adapters.py                   (token-overlap per domain)
  → scripts/publish_adapters.py                (HF Hub, auto model card)
  → src/ailiance_tuning/registry.py                (JSON registry in artifacts/)
```

## Hardware reality

- **KXKM-AI** (`kxkm@kxkm-ai`) is the ONLY training target. RTX 4090 24GB → Qwen2.5-32B QLoRA fits with `max_memory={0: "22GiB", "cpu": "50GiB"}` + `llm_int8_enable_fp32_cpu_offload=True`. Do not attempt locally on GrosMac.
- Eval + publish can run anywhere with disk (adapters are small: ~100-500 MB). Training runs are hours-to-days.
- Before launching a long training on kxkm-ai: **ask first** (see parent memory `feedback_no_launch_kxkm_without_ask.md`).

## Quick commands

```bash
# Tests (fast, no GPU)
uv run python -m pytest tests/ -v

# Dataset build + validate (cheap)
./scripts/build_all_datasets.sh                        # seeds only
./scripts/build_all_datasets.sh --with-hf              # + HF download
python scripts/validate_dataset.py datasets/processed/*.jsonl

# Single domain training (on kxkm-ai)
python scripts/train_sft.py \
  --base-model Qwen/Qwen2.5-32B-Instruct \
  --dataset datasets/processed/stm32_train.jsonl \
  --output-dir outputs/sft-stm32-qwen25-32b

# Eval all domains
python scripts/eval_adapters.py --samples 5

# Publish all to HF
python scripts/publish_adapters.py --org clemsail
```

## Gotchas

- **pyproject says Python >=3.12**, parent uses 3.14 — align with `uv` resolver, not system python
- **Test imports use `src.ailiance_tuning.*`** (not `ailiance_tuning.*`) — tests run from repo root, no `pip install -e .` required
- **`scripts.validate_dataset`** is imported by tests — scripts/ must stay importable (no top-level heavy deps)
- **Base model drift** : `scripts/eval_adapters.py` and `scripts/publish_adapters.py` hardcode `Qwen/Qwen3-8B`, but `scripts/train_sft.py` defaults to `Qwen/Qwen2.5-32B-Instruct`. When switching base, update all three + every YAML `base_model:` + recreate adapters (LoRA is base-specific)
- **Adapter path mismatch** : training writes to `outputs/sft-<domain>-qwen25-32b/`, eval expects `outputs/sft-<domain>/`. Symlink or rename before eval
- **ShareGPT vs OpenAI** : builders often produce ShareGPT (`conversations/from/value`), validator checks OpenAI (`messages/role/content`). Always run the `sharegpt_to_openai()` conversion before writing JSONL
- **HF streaming datasets** can hang without error — `--max-samples` is mandatory for reproducibility

## Domain catalog

10 domains: `stm32`, `kicad`, `embedded`, `platformio`, `iot`, `freecad`, `dsp`, `emc`, `power`, `spice` (plus `espidf` via `expand_espidf.py`). Each has its own system prompt and seed examples in `datasets/builders/build_<domain>_dataset.py`.

Prefix: `feat:`, `fix:`, `chore:`, `data:` (for dataset changes), `train:` (for recipe tweaks). Do **not** commit `outputs/`, `datasets/processed/`, `artifacts/*.json`, `.venv/`, `wandb/`.
